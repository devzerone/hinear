## Additional update: optimistic locking baseline

- `0004_add_version_for_optimistic_locking.sql` 추가
- domain `Issue`와 Supabase 타입에 `version` 필드 추가
- detail mutation route는 클라이언트가 읽은 `version`을 함께 받아 stale write를 감지
- repository update는 `id + version` 조건으로 수행되고, 불일치 시 `CONFLICT` 에러를 반환
- detail 화면은 `409`를 받으면 최신 issue를 다시 로드하고 conflict notice를 띄운다
- 다음 세션에서 Supabase MCP 읽기 호출로 auth 반영을 검증한 뒤 원격 Supabase에 `add_version_for_optimistic_locking` 마이그레이션 적용 완료
