## Implementation Notes

### MSW Handler Cleanup Pattern

```typescript
// In test file
describe("IssueDetailScreen", () => {
  afterEach(() => {
    server.resetHandlers(); // Clean up test-specific handlers
  });

  it("my test", async () => {
    server.use(customHandler); // Add temporary handler
    // ... test code
  });
});
```

### ConflictDialog Usage

```typescript
// In component
const [conflictInfo, setConflictInfo] = useState<{
  currentVersion: number;
  requestedVersion: number;
} | null>(null);

// On conflict
setConflictInfo({
  currentVersion: data.currentVersion,
  requestedVersion: data.requestedVersion,
});

// Render
{conflictInfo && (
  <ConflictDialog
    currentVersion={conflictInfo.currentVersion}
    requestedVersion={conflictInfo.requestedVersion}
    onDismiss={() => setConflictInfo(null)}
  />
)}
```

## Key Learnings

1. **useTransition in jsdom**: React의 동시성 기능이 jsdom 테스트 환경에서 호환성 문제를 일으킬 수 있음
2. **MSW handler management**: `server.use()`로 추가한 핸들러는 반드시 cleanup 필요
3. **Component composition**: 기존 atoms/molecules 활용해서 일관된 UI 유지
4. **Plan mode 활용**: 복잡한 문제 해결 시 체계적인 탐색과 계획 수립이 효과적
